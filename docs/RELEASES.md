# Release process

1. Update the application version and changelog.
2. Run `run_public_checks.ps1`.
3. Build a sanitized source archive with `python scripts/build_source_release.py`.
4. Test a clean clone or extracted archive in a fresh virtual environment.
5. Create a GitHub pre-release for preview versions.
6. Attach checksums and release notes.
7. Preserve profile data through explicit migrations.

## Release channels

- Stable: tested public releases
- Beta: opt-in feature and data testing
- Developer: internal or contributor diagnostics

The Windows executable should be generated by a documented GitHub Actions workflow after reproducible packaging and writable-data-path tests are complete.
